# Contributing

## Branching
- Branch from main
- Use short-lived branches
- Prefix branches with feat/, fix/, chore/, docs/, or refactor/

## Pull Requests
- All changes go through PRs
- Squash merge only
- No direct push to main

## Commit Messages
Use conventional commits:
- feat:
- fix:
- chore:
- docs:
- refactor:
- test:
- ci:

## AI-Assisted Code Rules
- AI-generated code must be manually reviewed
- Do not trust generated auth, access, or security logic without verification
- Refactor generated code when typing, structure, or maintainability is weak
